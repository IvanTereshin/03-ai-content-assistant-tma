# План разработки: AI Content Assistant TMA

## 1. Идея

Ассистент для владельцев Telegram-каналов. Пользователь выбирает задачу: придумать идеи, написать пост, улучшить текст, сделать контент-план или адаптировать длинный материал в короткий пост.

## 2. MVP

- Главная с быстрыми сценариями.
- Генератор идей.
- Генератор поста.
- Переписывание текста: короче, проще, экспертнее, продающее.
- Контент-план на неделю.
- История генераций.
- Черновики.
- Копирование результата и отправка в чат/канал.

## 3. Роли

- Автор: генерирует и сохраняет контент.
- Администратор: смотрит usage, шаблоны и лимиты.

## 4. Основные экраны

1. Главная: быстрые карточки задач.
2. Генератор: тема, аудитория, цель, тон, длина.
3. Результат: текст, варианты, действия.
4. Редактор: ручное редактирование и AI-команды.
5. Контент-план: календарь постов.
6. История: поиск по прошлым генерациям.
7. Настройки бренда: tone of voice, ниша, стоп-слова.

## 5. Данные

- `users`: telegram_id, name, plan, credits_left.
- `brand_profiles`: user_id, niche, audience, tone, banned_words.
- `generation_templates`: title, type, prompt_template.
- `generations`: user_id, template_id, input_json, output_text, tokens_used.
- `drafts`: user_id, title, content, status.
- `content_calendar_items`: user_id, draft_id, planned_date, status.

## 6. Backend API

- `GET /templates`
- `POST /generate`
- `POST /rewrite`
- `GET /generations`
- `POST /drafts`
- `PATCH /drafts/:id`
- `GET /calendar`
- `POST /calendar/items`
- `PATCH /brand-profile`

## 7. AI-сценарии

- Идеи постов по нише.
- Пост по краткому тезису.
- Перепаковка длинного текста.
- Хуки и заголовки.
- CTA для канала.
- Контент-план на 7 дней.
- Проверка текста на ясность.

## 8. Telegram-интеграция

- Bot menu button: "AI-ассистент".
- Авторизация через Telegram.
- Кнопка "Скопировать" в TMA.
- Кнопка "Отправить себе в чат" через backend и bot API.
- Опционально: share text в выбранный чат.

## 9. Технологии

- Frontend: React, Vite, TypeScript.
- Editor: textarea сначала, позже TipTap.
- Backend: Node.js + Fastify/NestJS.
- DB: PostgreSQL.
- Queue: BullMQ для долгих генераций.
- AI provider: OpenAI Responses API или совместимый провайдер.

## 10. Этапы

### Этап 1. Каркас и авторизация

- Создать TMA shell.
- Проверять `initData`.
- Сделать главную с задачами.

### Этап 2. Генерация

- Реализовать `/generate`.
- Добавить 5 первых шаблонов.
- Сохранять историю.

### Этап 3. Редактор и черновики

- Добавить экран результата.
- Добавить сохранение черновика.
- Реализовать AI-команды к выделенному тексту.

### Этап 4. Контент-план

- Создать календарь.
- Привязать черновики к датам.
- Добавить статусы: idea, draft, ready, posted.

### Этап 5. Ограничения и портфолио

- Добавить credits.
- Сделать demo mode с моковыми ответами.
- Подготовить красивые примеры генераций.

## 11. Критерии готовности

- Пользователь получает полезный текст за один сценарий.
- История генераций сохраняется.
- Черновик можно отредактировать.
- Контент-план показывает неделю.
- Ошибки AI-запросов обрабатываются понятно.
